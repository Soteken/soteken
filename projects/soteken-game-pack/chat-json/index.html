JSON от чата

Как сделать пак через обычный чат

Этот способ нужен, если нет Codex или MCP. Чат делает основу пака, а редактор потом помогает добавить медиа, проверить вопросы и экспортировать .siq.

  1. 1. Откройте любой чат

    Подойдет ChatGPT, Claude, DeepSeek или другой сервис, который умеет писать длинный текст и создавать файлы. В идеале чат должен вернуть файл soteken_pack.json.

  2. 2. Отправьте системную инструкцию

    Скопируйте этот prompt и отправьте его в чат перед своим запросом:

    Ты готовишь JSON-файл для импорта в SotekenGamePack.

Сначала уточни у пользователя:
1. Сколько раундов нужно?
2. Сколько тем в каждом раунде?
3. Сколько вопросов в каждой теме?
4. Какая стоимость вопросов в каждом раунде?
5. Для какой аудитории делаем пак?
6. Какие темы точно нужны?
7. Какие темы, шутки и форматы нельзя использовать?

После уточнений верни именно файл soteken_pack.json.
Не заменяй файл обычным объяснением.
Если сервис не умеет прикреплять файл, верни только JSON-код в отдельном блоке.

Структура JSON:
{
  "name": "Название пака",
  "author": "Автор",
  "rounds": [
    {
      "name": "Раунд 1",
      "themes": [
        {
          "name": "Тема",
          "questions": [
            {
              "price": 100,
              "text": "Вопрос",
              "answer": "Ответ"
            }
          ]
        }
      ]
    }
  ]
}

  3. 3. Попросите нужный пак

    Пример запроса:

    Сделай пак на 3 раунда, по 5 тем, по 7 вопросов.
Пак для друзей 20 лет. Темы: мемы, игры, кино, музыка, YouTube.
Не добавляй политику, жесткие 18+ темы и слишком локальные шутки.

  4. 4. Скачайте файл или скопируйте JSON

    ChatGPT и Claude часто могут создать файл. DeepSeek может показать JSON в отдельном окне с кнопкой скачивания. Если файла нет, скопируйте JSON-код и сохраните его как soteken_pack.json.

  5. 5. Импортируйте JSON в редактор

    Откройте SotekenGamePack, выберите импорт JSON от чата и загрузите файл. После импорта проверьте вопросы и добавьте картинки, аудио или видео.

Что важно проверить

Чат может повторять ответы, ошибаться в фактах или делать слишком длинные формулировки. Перед экспортом пройдитесь по всем вопросам и поправьте слабые места вручную.
